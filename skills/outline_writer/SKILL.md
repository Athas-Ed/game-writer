# 大纲写手技能

## 功能说明
- 输入：一句话剧情描述（例如“主角在图书馆发现一本神秘的书”）
- 输出：3-5个不同风格或走向的大纲方案，每个方案包含几段文字描述。
- 支持根据用户选择的方案进一步细化或修改。

## 工作流程
1. **读取设定**：调用 `scripts/read_settings.py` 汇总读取项目 `data/` 目录下的角色设定、背景设定等文件，将相关设定整合为上下文（内部实际复用通用工具 `src/tools/file_tools.py` 的 `read_settings_bundle()`）。
2. **生成多方案**：调用 `scripts/generate_outlines.py`，将一句话剧情 + 设定上下文 + 用户可能的偏好（如“偏向悬疑”、“注重感情线”）输入 LLM，生成多个方案。
3. **呈现方案**：将生成的方案以编号列表形式返回给用户，让用户选择。
4. **迭代细化**：用户选择后，根据选中的方案和提出的意见，再次调用 LLM 生成更详细的章节大纲、场景描述等。

## 使用示例
用户：“帮我写一个剧情大纲：主角在废弃车站发现一封旧信。”
Agent：
  1. 读取设定...（自动完成）
  2. 生成3个方案：
     方案A：悬疑向，信中提到失踪的列车员...
     方案B：温情向，信是多年前的恋人留下的...
     方案C：奇幻向，信能开启异世界入口...
  3. “请选择您喜欢的方案（输入A/B/C）”
用户：“A”
Agent：根据方案A生成详细大纲（约500字）...
